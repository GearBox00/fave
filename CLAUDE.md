# fave — 推し活記録アプリ

推し活の支出を「いくら使ってしまったか」ではなく「いくら推しに使えたか」という
ポジティブな視点で記録するアプリ。基本は家計簿だが、警告や罪悪感を与えるUIは避ける。

## 方針(ユーザー確認済み)

- スマホがメインの利用端末(スマホ幅優先のレイアウト)
- 推しは基本1人。複数推し対応は将来の有料機能候補(第1版では未実装)
- 「軍資金」機能あり: 月ごとに使っていい額を先に決め、枠内は罪悪感ゼロで使う設計
- ポジティブな文言を使う。ただし使いすぎを煽らない(軍資金がその歯止め)

## 構成

- 素のHTML/CSS/JavaScript(ビルド不要)
- データは localStorage(キー: `fave.v1`)に保存。書き出し/読み込みでJSONバックアップ可
- `index.html` / `style.css` / `app.js` が本体
- 画面: ホーム / きろく / りれき / せってい + 年間まとめ(ホームの「🎊 年間まとめを見る」から)
- 機能: 軍資金・マイルストーン・月別/年間グラフ・記念日カウントダウン・
  定期支出の自動記録(月替わりに自動追記)・まとめ画像シェア(Canvas生成、
  スマホは共有シート/PCはPNGダウンロード)
- `manifest.json` / `icon.svg` — スマホの「ホーム画面に追加」用
- `serve.js` — 動作確認用の簡易サーバー(`node serve.js` → http://localhost:8735)

## メモ

- グラフ・数字の濃いピンク `#e0487f` は白背景でのコントラスト検証済み。
  明るすぎる色(例 `#ff6b9a`)はグラフの棒には使わない
- **公開済み**: https://gearbox00.github.io/fave/ (GitHub Pages)
  - リポジトリ: https://github.com/GearBox00/fave (公開リポジトリ)
  - 更新手順: このフォルダで変更をコミットして `git push` すれば数分で反映される
- Service Worker(`sw.js`)でオフライン対応済み(stale-while-revalidate方式)。
  変更の反映は「次に開いたとき」なので、開発中は2回リロードが必要な場合がある。
  ファイルを追加したら `sw.js` の APP_SHELL にも追記すること
